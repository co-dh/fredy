### `effective_regular_additive_is_abelian` (line ~116, §1.594)

**Statement**: In an `EffectiveRegular + AdditiveCategory + HasZeroObject + HasEqualizers` category,
every monomorphism is a normal subobject (= kernel of some morphism).

**Root cause**: The standard proof constructs the **cokernel** of `m`:
```
coker(m) = coeq(m, zeroMorphism A B) : B → C
```
and then shows `m ≅ ker(coker(m))` using the effective-regular axiom.
`HasCoequalizers` is **absent** from the hypothesis list.

**Why `EffectiveRegular` alone doesn't substitute**:
- `EffectiveRegular.effective` takes an equivalence relation E on some object A and
  returns a cover `q : A → Q` with `E ≅ kernelPairRel q`.
- To use this for `m : A → B` monic, one would need to exhibit an equivalence relation
  on **B** whose "effectivization" produces the right cover.  The natural candidate is
  `E_m = BinRel.mk A m m hmp : BinRel 𝒞 B B`, but reflexivity of E_m requires
  `h : B → A` with `h ≫ m = id_B` — making m a split epi, which is not given.
- The kernel-pair relation `kernelPairRel m` on A is diagonal when m is monic
  (`kp₁ = kp₂`); effectiveness gives a cover `q : A → Q` with trivial kernel pair,
  hence q is monic-and-cover = iso — this just recovers `A ≅ Q`, not the desired `C`.

**Fix options**:

1. **Add `HasCoequalizers 𝒞` to the hypothesis** (minimal change):
   ```lean
   theorem effective_regular_additive_is_abelian
       (𝒞 : Type u) [Cat.{v} 𝒞]
       [EffectiveRegular 𝒞] [AdditiveCategory 𝒞] [HasZeroObject 𝒞]
       [HasEqualizers 𝒞] [HasCoequalizers 𝒞] :
   ```
   Then the proof is: `m = ker(coeq(m, zeroMorphism A B))`.
   This requires showing the cokernel of a monic equalizes m against 0.

2. **Derive `HasCoequalizers` from `EffectiveRegular + AdditiveCategory`**:
   In an effective regular additive category, coequalizers exist because every
   cover is a coequalizer of its kernel pair (`cover_is_coequalizer_of_level`)
   and regular epics = covers = coequalizers of their kernel pairs.
   This derivation is non-trivial and would live in a new section.

3. **Weaken the statement to abelian ⇔ with cokernels**:
   `AbelianCategory` already includes `HasEqualizers`; add `HasCoequalizers`
   and the proof goes through.
