## Sharpened blocker: `abelian_iff_regular_additive_all_normal`

This is NOT (as previously noted) a "cokernels-from-normality" calculus gap — `HasCoequalizers`
is already a hypothesis.  The true blocker is a **typeclass-coherence diamond** in the Lean encoding:

- `AbelianCategory` bundles `HasZeroObject`, `HasBinaryProducts`, `HasTerminal`, … as *fields*.
- The theorem's hypotheses supply `RegularCategory`, `AdditiveCategory`, `HasZeroObject`, … as
  *independent* instances whose shared parents (`HasTerminal`, `HasBinaryProducts`, …) are **not
  defeq**.  E.g. `reg.toHasTerminal ≠ zo.toHasTerminal`, `add.toHasBinaryProducts ≠ reg.toHasBinaryProducts`.
- Building `AbelianCategory` forces ONE coherent choice of terminal/products/coterminal; mixing the
  ambient instances triggers "field already specified" / `Type mismatch` on `prod_coprod_coincide`
  and `zero_eq_one`.
- The same mismatch breaks even the *trivial* (⟸) direction: `ab.all_normal` produces
  `IsNormalSubobject` w.r.t. the abelian category's *own* zero/equalizer instances, which differ
  from the ambient `[HasZeroObject]`/`[HasEqualizers]` appearing in the goal.

FIX OPTIONS (future work):
1. Reformulate the theorem with a single ambient bundle (e.g. take `[BicartesianCategory 𝒞]` or a
   pre-bundled "candidate-abelian" structure) so all parents are shared by construction.
2. Add propositional-coherence lemmas (`Subsingleton (HasTerminal 𝒞)`, etc.) and transport across them.
   Freyd never faces this because his categories carry a single ambient structure; the gap is purely
   an artifact of independent typeclass instances.

## `ExactCategory` strengthened (§1.597 faithfulness)

`ExactCategory.exact` now carries the canonical *factorization equation*
`cokernelMap (kernelMap x) ≫ θ ≫ kernelMap (cokernelMap x) = x` alongside `IsIso θ`.
Freyd §1.597 defines exactness via *this specific* coimage→image map being an iso, so the
equation is part of the data, not an afterthought.  Without it `monic_kernel_of_cokernel` is
unprovable (θ would be an *arbitrary* iso unrelated to x).  `ExactCategory` is used only in S1_59.
