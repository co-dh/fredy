## What is formalized (`Fredy/S1_74.lean`)

The abstract DOMINATION layer of §1.744, faithful to the book and free of any topological notion:

- `OntoObjects F` — the functor `F : D' → D` hits every object of `D`.
- `LeftFull F` — for each `A : D'` and each `g : F A ⟶ B`, a lift `f : A ⟶ A'` with `F A' = B` and
  `F f = g` (modulo that identification; encoded with the cast `hA' ▸ g`, since `F f` and `g` only
  share a type once `F A' = B`). This is Freyd's left-fullness verbatim ("for each `D ∈ |D'|` and
  each `g : F(D) → B` there exists `f : D → A` so that `F(f) = g`").
- `Dominates D' D` — `∃ F, OntoObjects F ∧ LeftFull F`.
- `dominates_refl`, `dominates_trans` — domination is reflexive and transitive. These are exactly
  the closure properties §1.745–§1.746 chain silently ("dominated by `P(D)`, hence by `D⁺`, hence by
  `ℕ*`, hence by `2*`").

Sorry count: **0**. The file imports only `Fredy.S1_18` (the `Functor` class) — no topology, no
sheaves. Domination is stated cross-universe (`u₁`/`u₂`/`u₃`) because §1.745–6 relate a category to
its tree of paths, a genuinely different type; the composite `Functor` is built by hand since the
repo's `compFunctor` instance is single-universe.
