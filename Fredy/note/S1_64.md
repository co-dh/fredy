## Key Blockers

### `amalgamation_lemma` (§1.651) and `pretopos_balanced` (§1.652)
Both require **disjointness of coproducts**: `inl(b) ≠ inr(c)` as a categorical axiom.
This is Freyd's *positivity* axiom (§1.62): coproducts are disjoint and universal.
Currently `PositivePreLogos` only has the bare coproduct universal property via `HasBinaryCoproducts`;
it does NOT include disjointness or universality. Adding these axioms would unblock:
- `amalgamation_lemma`: the equivalence relation E on B+C requires disjointness for transitivity
- `pretopos_balanced`: the cokernel-pair of a monic lives in B+B; disjointness forces monicity

Fix: add to `PositivePreLogos` the field
  `coprod_disjoint : ∀ {A B : 𝒞} (f : A ⟶ HasBinaryCoproducts.coprod A A), ...`
or use a dedicated `DisjointCoproducts` typeclass (disjointness + universality of coproducts).

### `monic_eq_cocover_preTopos` (§1.652)
Depends on `pretopos_balanced` and also has a specification issue:
the book's "monics = cocovers (= equalizers)" should use `HasEqualizers`, not `HasCoequalizers`.
`HEq ((HasCoequalizers.coeq p q).map) f` compares morphisms of different types via heterogeneous
equality — this is a placeholder that should be replaced with a proper equalizer statement.

### `prod_choice_is_choice` (§1.661) — **PROVED** (sorry-free; `Classical.choice` only)
The full §1.661 chain is now formalized on top of the (now-proven) intersection modular law
`modular_identity` (S1_56), via three reusable private lemmas in this file:

- `entire_comp_graph R hent p : Entire (R ⊚ graph p)` — post-composing an entire relation with
  the graph of a map stays entire (cover argument: pullback against `id` is iso, image-cover
  factors the left leg).  Supplies `f₁` (extract a map from `R ⊚ graph fst`, B₁ choice).
- `entire_refine R p f (hf : graph f ⊂ R ⊚ graph p) : Entire (R ⊓ (graph f ⊚ (graph p)°))` —
  the intersection-form §1.563 step.  From `modular_identity` (R, graph p, graph f) plus
  `graph f ⊂ R⊚graph p` we get `graph f ⊂ R'⊚graph p`; its witness section makes `R'.colA`
  a composite of covers, i.e. `Entire R'`.  (`R` need not be entire — totality is carried by `f`.)
- `comp_recip_pin f p : (graph f ⊚ (graph p)°).colA ≫ f = (graph f ⊚ (graph p)°).colB ≫ p` —
  the "agree at the shared object" pinning, descended along the image-cover (covers are epic).

Assembly: with `R' := R ⊓ (graph f₁ ⊚ (graph fst)°)` entire and pinned (`R'.colB ≫ fst =
R'.colA ≫ f₁`), the pair `⟨R'.colA, R'.colB ≫ snd⟩` is jointly monic (pinning makes the fst-leg
redundant), so the B₂-relation `R'₂` is entire; `Choice B₂` yields `f₂` with a **single** witness
`w : A → R'.src`.  Pinning gives `w ≫ R'.colB = pair f₁ f₂`, and `R' ⊂ R` carries `w` into `R.src`.

### `preTopos_boolean_iff_all_decidable` (⇐)
Requires pullback-stability of complements: if `K : Subobject 𝒞 C` is complemented and
`f : B → C`, then `InverseImage f K` is complemented.  This in turn needs:
- `entire_B ≤ InverseImage f (union K K')` ← need `InverseImage f (entire_C) ≅ entire_B`
  (pullback of id_C along f is id_B) and monotonicity of InverseImage.
- The key step "every subobject is a pullback of the diagonal" (hence complemented) requires
  `monic_eq_cocover_preTopos` (every monic is an equalizer), which depends on `pretopos_balanced`.
So `⇐` is transitively blocked by the missing positivity/disjointness axiom.

### Session log — coproduct-disjointness pass (this session)

**Question answered: is coproduct disjointness a book DEFINITION or a DERIVED theorem?**
→ It is **DEFINITIONAL** (a defining property of a *positive* pre-logos), NOT derivable from
the bare `HasBinaryCoproducts` universal property.  The book is explicit:

- **§1.623** "A POSITIVE PRE-LOGOS is a pre-logos in which for every pair of objects A, B there
  exists [a coproduct] … We can choose C so that A ∪ B = C."  The coproduct is *constructed* as
  the ambient object in which A, B sit as subobjects with `A ∪ B = C`.
- **§1.621** "If A₁, A₂ are subobjects of A in a pre-logos such that `A₁ ∩ A₂ = 0` and
  `A₁ ∪ A₂ = A` then A is a coproduct of A₁, A₂."  So a positive coproduct IS a disjoint
  (`∩ = 0`), complemented (`∪ = whole`) union — disjointness is part of the data.
- **§1.626** "Coproducts can exist without positivity.  Any distributive lattice … is a pre-logos
  with coproducts.  It is positive iff it is degenerate."  ⇒ the bare universal property does
  **not** imply disjointness (in a lattice `inl`, `inr` are not jointly monic and `A∧B ≠ 0`).
  This is the proof that we must ADD disjointness as structure, not derive it.
