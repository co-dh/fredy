## Closed this pass

- **`membershipMap_nameOf`** (§1.94): `membershipMap (nameOf m hm) = classify m hm`.
  Pure β-reduction: `pair id (term ≫ curry g) = pair id term ≫ (A × curry g)`,
  then `(A × curry g) ≫ eval = g = fst ≫ χ_m` and `pair id term ≫ fst = id`.
  Uses only `curry_eval_eq`, `prodMap_fst/snd`, `pair_uniq`.

- **`inter_le_named`** (§1.943 pt1): given `'A'' = F_name`, `membershipMap_nameOf`
  rewrites `membershipMap F_name = χ_{A'}`, so `∩F = InverseImage(χ_{A'}, true)`.
  The classify pullback (`classify_pullback`, S1_9) says A' is itself the pullback
  of `true` along `χ_{A'}`; its universal property applied to the *chosen* pullback
  cone yields the factorizing map `∩F.dom → A'.dom`.  The classify pullback axiom
  exists in `HasSubobjectClassifier` — no missing infrastructure.

- **`singletonMap`** (§1.94(10)): `curry (classify (diag A) (diag_mono A))`, i.e.
  `curry(χ_{Δ_A})`.  Now a real definition (was a placeholder sorry).

## Note on `sorryAx` in closed proofs

`lean_verify` reports `sorryAx` for `inter_le_named` / `membershipMap_nameOf` /
`singletonMap`, but **not** because of these proofs — they contain no `sorry`.
The axiom is inherited transitively through `topos_has_exponentials` (S1_92:39),
which is the topos→`HasExponentials` instance and is itself a `sorry` stub. Any
S1_94 declaration mentioning `powObj`/`curry`/`eval_exp` carries that axiom. The
proofs here are logically complete given the exponentials instance; closing
S1_92's instance would clear the axiom with no change to S1_94.
