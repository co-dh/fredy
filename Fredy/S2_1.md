## Integrity note: SemiSimple recip direction (fixed)

The original `SemiSimple` encoded `R = F ≫ G°` (F : a→c, G : b→c, common *target*).
That is the WRONG reciprocal: Freyd §2.16(10) defines a semi-simple morphism as `R = F°G`
with F, G simple, and §2.143 fixes the typing — a tabulation `(F, G)` of R has F, G sharing a
common *source* apex, giving `R = F° ≫ G` (F : c→a, G : c→b). Simplicity (`F°F ⊑ 1`) is not
preserved under reciprocation, so `F ≫ G°` with F,G simple is a genuinely different (and
non-book) predicate. Fixed to `R = F° ≫ G`. Downstream `S2_3.UnionOfSemiSimple` already
quantified over `F° ≫ G` directly, so it was unaffected; `S2_4.pre_positive_semi_simple_iff_metonymic`
and `S2_2` only mention `SemiSimple`/`SemiSimpleAllegory` at the type level. No proof relied on
the wrong encoding; no sorry added or removed.
