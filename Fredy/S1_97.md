## Sorry reasons

- **nno_is_coproduct**: The inverse `inv = iterate inl (case(zero≫inr)(succ≫inr)) : N → 1+N`
  can be constructed and `inv ≫ [0,s] = id_N` proved (by NNO uniqueness after showing
  `case(zero≫inr)(succ≫inr) ≫ [0,s] = [0,s] ≫ succ` via case_uniq).  However, the other
  direction `[0,s] ≫ inv = id_{1+N}` reduces (via case_uniq) to showing `succ ≫ inv = inr`,
  which requires `[0,s]` to be monic — a circularity that Freyd resolves via the Peano property
  (§1.987) and §1.988.  No shorter proof path exists at this axiom level.

- **nno_peano_property**: Requires §1.986 infrastructure (reflexive-transitive
  closure of relations, §1.947) which is not yet in the repo.

- **least_peano_subobject**: Same dependency on §1.947 relational calculus.

- **nno_of_bicartesian_data / bicartesian_functor_preserves_nno**: The proof in
  the book cites [2.542] (boolean covers) and uses the Peano property from §1.988.

- **ac_iff_iac_and_projective_one**: Needs the pullback-along-f argument from §1.974
  plus well-supportedness infrastructure (§1.61).

- **free_action_iff_bicartesian**: Analogue of §1.98(10); same dependency on Peano/§1.988.

- **free_action_exists**: In a topos with NNO, the free A-action exists as the
  "list object" Σ_{n:N} A^n.  Constructing this requires the NNO + exponentials +
  a colimit construction (or primRec over pairs), which is beyond what is currently
  formalised in the repo.

### `nno_is_free_one_action` (§1.98(12))

**Construction.** The NNO (N, 0, s) is a free 1-action with:
- `obj = N`, `unit = zero`, `act = snd ≫ succ : 1 × N → N`
- `recA α := hN.iterate α.unit (pair(term α.obj)(Cat.id α.obj) ≫ α.act)` = NNO iterate
  with step `f_rec α := pair(term,id) ≫ α.act : α.obj → α.obj` ("apply α.act to (*, x)").

**Key identity** (helper `prodMap_factorN`):
`prodMap one N B h = (snd ≫ h) ≫ pair(term B)(Cat.id B)`
proved by `pair_uniq`: fst component = `term` (by `term_uniq`), snd component = `snd ≫ h` ✓.

**recA_unit**: direct from `hN.iterate_zero`.

**recA_act**: `prodMap one N B iter ≫ α.act`
= `((snd ≫ iter) ≫ pair(term,id)) ≫ α.act` [prodMap_factorN]
= `snd ≫ iter ≫ (pair(term,id) ≫ α.act)` [Cat.assoc × 3, congr 1]
= `snd ≫ succ ≫ iter` [← iterate_succ with explicit args]
= `(snd ≫ succ) ≫ iter` [Cat.assoc].

**recA_uniq**: from hypothesis `hms : prodMap one N α.obj m ≫ α.act = (snd ≫ succ) ≫ m`,
applying `prodMap_factorN` and reassociating gives
`snd ≫ m ≫ f_rec α = snd ≫ succ ≫ m` (as maps `1 × N → α.obj`).
Cancel `snd` from the left using its section `prodOneLeftInv N` (which satisfies
`prodOneLeftInv N ≫ snd = id_N`, i.e., `snd` has a section, so is left-cancellable).
Conclude `m ≫ f_rec α = succ ≫ m`, then `hN.iterate_unique` closes the goal.

### `nno_terminal_is_coequalizer` (§1.985)

Given `f : N → X` with `succ ≫ f = f`, take `g = zero ≫ f`.
Uniqueness: `zero ≫ term N = id₁` by terminal uniqueness.
Existence: both `f` and `term N ≫ g` satisfy NNO equations for `(g, id_X)`,
so NNO uniqueness gives `f = term N ≫ g`.
