## Proved Declarations (non-trivial)

- `terminator_is_choice` — §1.661: `one` is always choice in a regular category.
  Proof: any entire `R : BinRel 𝒞 A one` is automatically simple by terminal
  uniqueness (`term_uniq`), hence a map, so `R.colA` is iso and its inverse
  is the required section.

- `coprod_choice_to_one_one_choice` — §1.662 (1)→(2): immediate from `terminator_is_choice`.

- `subobject_of_choice_is_choice` — §1.66: a monic-subobject `m : A ↣ C` of a choice
  object `C` is choice.  Proof (axiom-free, no `⊚`-algebra): given entire `R : X → A`
  with tabulation ⟨src; colA, colB⟩, post-compose the right leg with `m` to get
  `R' : X → C` (= ⟨src; colA, colB ≫ m⟩; the pair stays jointly-monic because `m` is
  monic).  `R'` has the SAME left leg `colA`, a cover, so `R'` is entire.  `C` choice
  yields a witness `h : X → src`; the same `h` witnesses the map `h ≫ colB : X → A`
  inside `R`.  Monicity of `m` is used only to keep `R'` a valid relation.

- `quotient_of_choice_is_choice` — §1.66: a cover target `x : C ↠ B` of choice `C`
  is choice.  Proof (axiom-free): given entire `R : X → B` (left leg `colA` a cover),
  pull the cover `x` back along the right leg `R.colB`; the pulled-back leg `π₂` into
  `R.src` is again a cover (`cover_pullback`).  Form `R'' : X → C` = ⟨pb.pt; π₂ ≫ colA,
  π₁⟩, whose left leg `π₂ ≫ colA` is a composite of covers (`cover_comp`), so `R''` is
  entire; joint-monicity comes from the pullback's `lift_uniq`.  `C` choice gives a
  witness `h`; then `h ≫ π₂ : X → R.src` witnesses the map `h ≫ π₁ ≫ x : X → B` in `R`.

### `amalgamation_lemma` (§1.651)

Need: every equivalence relation on B+C has a coequalizer (= quotient).

The cokernel-pair of a monic m lives in B+B; monic+epic forces m to split.

### `prod_choice_is_choice` (§1.661)
CONSTRUCTIVE HALF NOW DONE (sorry-free): `choice_factor_through_map` (private, this file)
extracts both single-factor maps `f₁ : A → B₁`, `f₂ : A → B₂`.  For projection map
`g : C → D` it forms the image relation `R_g = {(R.colA a, (R.colB≫g) a)}` ⊆ A×D, proves
it jointly monic, shows its left leg is a cover (it post-factors the cover `R.colA`, using
"right factor of a cover is a cover"), hence entire, then `Choice D` returns the factor map
with its section over `R_g.src`.  Verified axioms: only `Classical.choice` (no `sorryAx`).

REMAINING GAP (the one `sorry`): the two factor maps come with INDEPENDENT witness sources
(`R_fst.src`, `R_snd.src`), each above `A` but NOT above `R.src`.  Gluing them into one
witness `w : A → R.src` realizing ⟨f₁,f₂⟩ inside `R` is the book's "R ∩ (f₁∘fst°) is entire"
step — the *intersection* modular law §1.563: `RS ∩ T ⊂ (R ∩ T S°) S`.

BLOCKER SHARPENED: this repo only has the *associativity* form of the modular law,
`modular_identity` ((R⊚S)⊚T° ⊂ R⊚(S⊚T°), Fredy/S1_56.lean, open `sorry`).  The
*intersection* form RS ∩ T ⊂ (R ∩ T S°) S — and the corollary "entire ∩ functional-graph
stays entire" that the gluing actually needs — is NOT EVEN STATED in S1_56.  The book proves
it only via the §1.55 Henkin–Lubkin representation (also not formalized).  To close
`prod_choice_is_choice` one must first (1) state and prove the intersection modular identity
in S1_56 (needs the §1.55 representation), then (2) derive `Entire (R ⊓ graph f₁∘fst° form)`,
then (3) project to B₂.  Out of scope for S1_64 alone; faithful `sorry` retained.
