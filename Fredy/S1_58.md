## Key gaps and how to close them

### `middle_two_interchange`
The book's proof uses TWO binary operations on Hom(A,B): left addition (+_L, via coproduct diagonal) and right addition (+_R, via product codiagonal). It proves `(u +_R v) +_L (x +_R y) = (u +_L x) +_R (v +_L y)` from universality, then shows +_L = +_R. The current `HalfAdditiveCategory` has only one `add` field; to prove interchange, expose both additions from `prod_coprod_coincide` and derive equality first.

### `image_via_coeq`
Need: `q : A → C` (coequalizer of kp₁, kp₂) is a cover (in a regular cat, any coequalizer-of-its-kernel-pair cover is epic by §1.512); then `x = q ≫ m` is an epi-mono factorization; uniqueness of the image gives `m` monic. Requires `cover_is_coeq_is_cover` (converse of §1.566).

### `bicart_repr_preserves_covers`
Two sub-lemmas needed: (1) given a propositional coequalizer UMP for f (from §1.566), construct a `HasCoequalizer` instance; (2) in a regular category, if `q` is a `HasCoequalizer` instance, then `q.map` is a cover. Neither is in the library yet.
