# ¬ß1.57 ‚Äî Pullbacks, equalizers, kernel pairs

Notes supporting `Freyd/S1_57.lean`. Composition in **diagram order**: `m;e` means first `m`
then `e` (Freyd's convention).

## Are pullback legs monic?

A pullback is a square, not a map; the question is about its **legs** (projections). In general
**no** ‚Äî e.g. a product `A √ó B ‚Üí A` is a pullback over the terminal object, and it is not monic.

Two special cases matter:

- **Mono is stable under pullback.** Pulling back a monic `m` along anything makes the *opposite*
  leg monic. (Freyd: "left-invertibles are always transferred by pullbacks"; the same holds for
  monics.)
- **Equalizer maps are always monic** ‚Äî this is `eqMap_mono` (`Freyd/S1_57.lean:145`). It follows
  from the previous point (see below).

## Equalizer vs. kernel pair

"A pullback with the same `x` on both legs" is the **kernel pair**, *not* the equalizer.

### Kernel pair

Kernel pair of `x : A ‚Üí B` = pullback of `x` against **itself** (same map both legs):

```
ker x ‚îÄ‚îÄkp‚ÇÇ‚îÄ‚îÄ‚Üí A
  ‚îÇ            ‚îÇ
 kp‚ÇÅ           x
  ‚Üì            ‚Üì
  A ‚îÄ‚îÄ‚îÄ‚îÄx‚îÄ‚îÄ‚îÄ‚îÄ‚Üí B
```

This is the *level* of `x` from ¬ß1.571. Its two legs `kp‚ÇÅ, kp‚ÇÇ` are **not** individually monic ‚Äî
they are only *jointly* monic (a tabulation / monic pair).

### Equalizer

Equalizer of `f, g : A ‚Üí B` is a different limit. It can be written as a pullback, but against the
**diagonal**, not "the same map twice":

```
eq(f,g) ‚îÄ‚îÄ‚îÄ‚Üí B
   ‚îÇ         ‚îÇ
   m         Œî = ‚ü®id,id‚ü©
   ‚Üì         ‚Üì
   A ‚îÄ‚îÄ‚ü®f,g‚ü©‚Üí B√óB
```

The equalizer map `m` is the leg over `A`.

**Source of `‚ü®f,g‚ü©` is `A`, not `A√óA`.** The pairing combines two maps that share a *domain* (`A`)
into the *product of their codomains* (`B√óB`): `‚ü®f,g‚ü©(a) = (f a, g a) : A ‚Üí B√óB`. The product `B√óB`
sits on the *target* side. (`A√óA` would arise only from `f√óg : A√óA ‚Üí B√óB`, the product of two maps
out of different objects ‚Äî a different operation.)

**Why `m` is monic:** `Œî : B ‚Üí B√óB` is split monic (left inverse `œÄ‚ÇÅ`), and monos are
pullback-stable, so its pullback `m` is monic. That is the conceptual reason behind `eqMap_mono`.

### The equalizer pullback square in Set

In **Set**, the pullback of two functions `u : A ‚Üí C` and `v : B ‚Üí C` is

```
{ (a,b) ‚àà A√óB : u(a) = v(b) }
```

For the equalizer square take `u = ‚ü®f,g‚ü© : A ‚Üí B√óB` and `v = Œî : B ‚Üí B√óB`:

```
eq(f,g) = { (a,b) ‚àà A√óB : (f a, g a) = (b, b) }
        = { (a,b) : f a = b  and  g a = b }
```

The condition forces `f a = g a` (both equal `b`), and then `b` is determined as that common value.
So the pullback set is in bijection with the **equalizer subset**

```
E = { a ‚àà A : f a = g a }
```

via `(a,b) ‚Ü¶ a` (with `b = f a = g a` recovered automatically).

The two legs under this identification:

- **`m : eq(f,g) ‚Üí A`** ‚Äî left projection `(a,b) ‚Ü¶ a`, i.e. the **subset inclusion** `E ‚Ü™ A`. It is
  injective (a subset inclusion never repeats elements), which is why `m` is monic.
- **top leg `eq(f,g) ‚Üí B`** ‚Äî right projection `(a,b) ‚Ü¶ b`. Since `b = f a = g a`, this is the
  **common-value map** `a ‚Ü¶ f a (= g a)`: the restriction of `f` (equivalently `g`) to `E`.

Commutativity: for `a ‚àà E`, `m` then `‚ü®f,g‚ü©` gives `(f a, g a) = (f a, f a)`; top-leg then `Œî` gives
`Œî(f a) = (f a, f a)`. Equal. ‚úì

Concrete example: `A = {1,2,3}`, `B = {x,y}`, `f : 1‚Ü¶x, 2‚Ü¶y, 3‚Ü¶x`, `g : 1‚Ü¶x, 2‚Ü¶x, 3‚Ü¶x`. They disagree
only at `2`, so `E = {1,3}`; `m` is the inclusion `{1,3} ‚Ü™ {1,2,3}` and the top leg is `1‚Ü¶x, 3‚Ü¶x`.

## How they connect in ¬ß1.571

The proof `ac_factorization_via_idempotent` uses both, in different roles:

- `kernelPairRel x` / `kernelPairRel e` ‚Äî pullback of a map against itself = the **level**. Legs
  jointly monic, not monic. Used to state "same level."
- `eqMap (id_A) e` ‚Äî the **equalizer** of `id` and `e`, i.e. pullback of `‚ü®id,e‚ü©` against `Œî`. This
  `m : C ‚Üí A` *is* monic, which is exactly what makes the second leg `n = m;x` monic.

So: kernel pair = "same map on both legs," legs not monic. Equalizer = pullback of the diagonal,
leg always monic. Different constructions, both pullbacks.
