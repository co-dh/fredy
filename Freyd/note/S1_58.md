- `add_eq_addL : add x y = diag ≫ Φ⁻¹ ≫ case x y` — the coproduct/codiagonal operation `+_L`.
- `add_eq_addR : add x y = pair x y ≫ Φ⁻¹ ≫ case id id` — the product/diagonal operation `+_R`.

`add_zero`/`zero_add` are now **proven theorems** (in namespace `HalfAdditiveCategory`), not fields.
The interchange is Freyd's Eckmann–Hilton argument, fully constructive given the iso witness:

1. `case_comp`, `comp_pair` — post/pre-composition collapse a `case`/`pair` (coproduct/product
   functoriality).
2. `case_pair_swap` — the matrix identity `case (pair a b)(pair c d) = pair (case a c)(case b d)`,
   pure (co)product universality, no iso. This is the heart of the argument.
3. `middle_two_interchange` — read the LHS with outer `+_L`/inner `+_R` and the RHS with outer
   `+_R`/inner `+_L`; both reduce to the single composite
   `diag ≫ Φ⁻¹ ≫ pair (case u x)(case v y) ≫ Φ⁻¹ ≫ ∇`, equal by `case_pair_swap`.

## §1.587 explained (the programmer version)

**The setup.** A *Diophantine equation* is a polynomial equation where you only accept whole-number
answers, like `x*x = 2*y*y + 1`. The question: *does it have a solution?*

**Fact 1 (easy half).** You can write a program that finds a solution **if one exists** — just loop
over all tuples and test:

```python
for x, y, ... in all_tuples():   # 0,1,2,... in every slot
    if p(x,y,...) == q(x,y,...):
        return "SOLVABLE"
# if no solution: loops forever
```

**Fact 2 (hard half).** If there's **no** solution, that loop runs forever — and there is *no*
smarter program that always halts and says "no solution." This is Hilbert's 10th problem, proven
impossible in 1970 (MRDP). Same wall as the halting problem: you can confirm a "yes," never reliably
confirm a "no." So **"solvable" is checkable-eventually; "unsolvable" can never be listed.**

**Freyd's trick.** Category theory can *say* "this equation has no solution" using only products,
coproducts, and equalizers. The **equalizer** of `p` and `q` is exactly the *set of solutions*:

- has a solution → equalizer is non-empty
- no solution → equalizer is **empty** (a coterminator)

So the categorical sentence **"this equalizer is empty"** is true (in the world of sets) **exactly
when the equation is unsolvable.** `E` is not always empty — `E` depends on `p,q`; it *is* the
solution set `{ x : p(x) = q(x) }`.

**The punchline.** Since "unsolvable" can never be listed by any program, the set of true categorical
sentences can never be listed either. Meaning: **no finite rulebook / axiom system captures all the
truths about sets in this language** (an axiom system only ever generates an enumerable list of
theorems). Arithmetic's undecidability leaks into category theory through nothing fancier than
equalizers.

To pin ℕ down in the first place, Freyd uses a *natural-number object*: `1 →⁰ N →ˢ N` with
`1 + N →^{[0,s]} N` an **isomorphism** (every number is 0 or a unique successor) and `N → 1` a
**coequalizer** of `1_N, s`. Then `+` and `×` come from the primitive-recursion diagrams, so every
polynomial becomes a morphism `Nᵏ → N` — and the trick above applies.
